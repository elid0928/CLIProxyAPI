## Codex 凭证检查

### 需求描述

程序启动后, 通过配置, 设置 Codex 凭证有效检查间隔时间, 定时检查 Codex 凭证是否过期, 如果出现额度小于10% 或者401错误, 则直接删除凭证, 以免后续请求继续使用过期的凭证导致请求失败

### 设计方案
1. 在程序启动时, 读取配置文件中的 Codex 凭证检查间隔时间, 并启动一个定时器, 定时执行 Codex 凭证检查任务。
2. Codex 凭证检查任务会调用 Codex API 获取当前凭证的使用情况, 包括剩余额度和过期时间。
3. 如果发现凭证的剩余额度小于10% 或者 API 返回401错误, 则立即删除该凭证, 并记录日志以便后续分析。
4. 定时器会继续运行, 定期检查凭证的状态, 确保程序始终使用有效的 Codex 凭证进行请求。
